Working With Octopress

Oct 23rd, 2013

My work process. And some pointers on where to get info on these things. Pretty simple.

  • Edit
  • Generate
  • Preview
  • Commit
  • Deploy
  • Repeat…

Edit

New Posts and Pages

Use rake new_post["title"] or rake new_page["path or path/title"] and then edit the file that’s created.

http://octopress.org/docs/blogging/

Header, Footer, Navigation Updates

These are sections of the site that are most likely to be customized. You can edit each html file in /source/_includes/custom/ and your changes will be preserved across updates.

http://octopress.org/docs/theme/template/

Site Configurations

It’s all about the _config.yml. It’s in the root of your octopress directory. There are three main sections to it.

  • Main: Settings typical of html pages headers and footers
  • Jekyll: All settings coming from Jekyll and define the way the site is built
  • Third Party: octopress comes with a set of predefined links to external services, like Twitter and Facebook. In this section you can setup basic access settings to these services.

http://octopress.org/docs/configuring/

Generate

rake generate generates your blog based on the edits you just made

Preview

rake preview is pretty cool. Run it and then navigate to http://localhost:4000 to see your changes locally. Sure beats waiting for the site to sync up remotely.

Commit

Like it? Commit it. The basics from the root of your octopress dir:

git add .
git commit - m "your commit comment"
git push

Deploy

rake deploy copy the generated files into _deploy/, add them to git, commit and push them up to the master branch. I’m using github, more on that here: http://octopress.org/docs/deploying/github/

Repeat
